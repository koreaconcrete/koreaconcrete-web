# Local Test Checklist

## 접속 URL

- 프론트엔드: `http://localhost:8080/index.html`
- 상품 상세 예시: `http://localhost:8080/product-detail.html?id=1`
- 관리자 상품 수정 예시: `http://localhost:8080/admin-product-form.html?id=1`
- Swagger UI: `http://localhost:8080/swagger-ui/index.html`
- API Base URL: `http://localhost:8080/api/v1`

## 공통 준비

1. 백엔드 실행:
   ```bash
   export SPRING_PROFILES_ACTIVE=dev
   export JWT_SECRET=replace-with-at-least-32-random-characters
   ./gradlew bootRun
   ```
2. 프론트엔드는 별도 실행하지 않고 백엔드에서 제공되는 `http://localhost:8080/index.html`로 접속한다.
3. seed 계정:
   - 관리자: `admin` / `Password1234!`
   - 일반회원: `buyer` / `Password1234!`

## Public 기능 점검

### 메인

- `index.html` 접속 시 주요 카테고리와 추천 상품이 보인다.
- 검색어 입력 후 상품 목록으로 이동한다.
- 인기검색어 영역이 비어 있어도 기본 키워드가 보인다.

### 상품 목록

- 상품 카드가 3개 seed 상품 기준으로 표시된다.
- 검색어 `그레이팅`, `수로관`, `경계석`으로 필터링된다.
- 카테고리 필터 선택 시 해당 카테고리 상품만 조회된다.
- 카테고리/정렬 필터 변경 시 목록이 즉시 갱신되고 URL에도 필터가 반영된다.
- 빈 결과일 때 empty 상태가 표시된다.

### 상품 상세

- 제목/요약 아래에 대표 이미지가 표시되고, 이미지가 잘리지 않는다.
- 추가 설명 이미지는 `제품 이미지 및 규격표` 섹션에 표시된다.
- 카테고리, 요약, 규격 select가 표시된다.
- 규격 선택 시 가격/VAT/운반비 정책이 갱신된다.
- 수량 입력 후 바구니 버튼을 누르면 담김 모달이 표시되고, 모달에서 바구니로 이동할 수 있다.
- 품절 규격은 바구니 담기/견적요청 버튼이 비활성화된다.
- 로그아웃 상태에서 바구니 담기/견적요청 버튼을 누르면 로그인 화면으로 이동한다.
- 운반비 예상 계산에서 주소와 차량을 넣으면 금액과 안내 문구가 표시된다.
- 문자상담/전화상담 요청 시 이름/연락처 입력 후 접수된다.

### 바구니

- 담은 상품 이미지, 설명, 규격, 수량, 단가, 상품금액이 표시된다.
- 상품 행을 클릭하면 상품 상세로 이동한다.
- 수량 변경 시 localStorage 품목이 갱신된다.
- 수량 변경 시 상품금액과 전체 총합이 다시 계산된다.
- 가격 미등록 품목은 `견적문의`로 표시되고 합계 안내 문구가 보인다.
- 삭제 버튼으로 품목을 제거할 수 있다.
- 견적요청으로 이동한다.

### 견적요청

- 회사명, 담당자, 연락처, 현장주소, 희망납기, 메모를 입력한다.
- 개인정보 동의 없이는 제출되지 않는다.
- 품목 목록에 상품 이미지와 설명이 표시되고, 상품 클릭 시 상세로 이동한다.
- 제출 성공 시 요청번호가 표시되고 장바구니가 비워진다.

### 회원

- 회원가입 후 JWT가 저장되고 마이페이지로 이동한다.
- 로그인 성공 후 이전 페이지 또는 메인으로 이동한다.
- 마이페이지에서 내 정보와 내 견적 목록을 확인한다.

## Admin 기능 점검

### 로그인

- `admin`으로 로그인한다.
- 관리자 페이지 접근 시 대시보드 API가 성공한다.
- 로그아웃 또는 토큰 삭제 후 관리자 페이지 직접 접근 시 로그인 화면으로 이동한다.
- 일반 회원이 관리자 페이지에 접근하면 권한 안내가 표시된다.

### 대시보드

- 오늘 견적 수, 오늘 상담 수, 전체 상품 수가 표시된다.
- 최근 견적/상담 목록이 표시된다.

### 카테고리 관리

- 카테고리 트리가 테이블로 표시된다.
- 새 카테고리를 생성하면 목록에 추가된다.
- 상위 카테고리를 드롭다운에서 선택하면 하위 카테고리로 생성된다.

### 상품 관리

- 상품 목록, SKU, 상품명, 카테고리, 상태가 표시된다.
- 상태 필터로 판매중/숨김/품절 등 상품을 분리 조회한다.
- 상품명뿐 아니라 관리자에서 등록한 검색어로도 상품을 필터링한다.
- 수정 버튼으로 상품 입력 화면에 진입하면 기존 값이 채워진다.
- 대표 이미지/설명 이미지를 업로드하면 이미지 목록과 미리보기에 즉시 반영된다.
- 이미지 순서를 위/아래로 변경하고 삭제할 수 있다.
- 규격을 추가하고 규격명, 폭/길이/높이/두께/중량/단위/상태를 수정할 수 있다.
- 규격별 판매단가를 생성/수정할 수 있다.
- 상품 카드와 상세 상단 미리보기가 입력값에 맞춰 갱신된다.
- 상품 저장 후 `상품, 규격, 가격이 저장되었습니다.` 메시지가 표시된다.

### 가격 관리

- 가격표 목록이 표시된다.
- 상품 수정 화면에서 규격별 가격을 생성/수정할 수 있다.
- API로 `POST /api/v1/admin/product-prices`, `PATCH /api/v1/admin/product-prices/{priceId}`가 동작한다.
- 상품과 variant가 일치하지 않으면 실패한다.

### 운반비 관리

- 지역/차량/기본 운반비를 입력해 FreightRateRule을 생성한다.
- 상품 상세 운반비 계산에서 해당 지역이 반영된다.
- 룰이 없으면 fallback 300,000원과 관리자 확인 안내가 표시된다.

### 견적 관리

- 견적 목록에서 상태 필터가 동작한다.
- 상세 화면에서 고객 정보와 품목을 확인한다.
- 상세 화면에서 품목 이미지와 상품 요약을 함께 확인한다.
- 상태를 변경하고 저장한다.

### 상담 관리

- 상담 목록에서 상태 필터가 동작한다.
- DONE 버튼으로 상담 상태를 완료 처리한다.

### 회원 관리

- 회원 목록, 아이디, 이름, 권한, 상태가 표시된다.

## API 보안 점검

- `GET /api/v1/quotes/{id}`는 로그인 없이 접근 거부된다.
- 로그인한 회원은 자기 견적만 조회 가능하다.
- 잘못된 JWT는 401을 반환한다.
- 관리자 API는 `ROLE_ADMIN`, `ROLE_OPERATOR`, `ROLE_PRODUCT_MANAGER` 정책에 따라 접근이 제한된다.
- 상품과 variant가 다른 요청은 400으로 거부된다.
- 수량 0 이하 요청은 400으로 거부된다.

## 제외 기능 확인

- 자료실/게시판/공지사항/FAQ/도면자료/물가자료/카탈로그 게시판 메뉴가 없다.
- `Board`, `Post`, `FileAsset`, `BoardKey`, `FileOwnerType` 엔티티가 없다.
